02
Unified Schema

Pydantic Event Model & Avro Schema Definition

Every platform speaks a different schema. Define the canonical FeedbackEvent once. Validate with Pydantic at ingestion time. Serialize to Avro for Kafka. Any event that fails validation goes to the DLQ — never silently dropped.

🔥
Production Issue #1: Platform Schema Mismatch at 3 AM Google Maps changed rating from an integer (1–5) to a float (4.6) in their API v3 response without announcement. Our Pydantic model expected int, threw a ValidationError for every Google review, and the connector silently dropped 800 reviews before the on-call alert fired. Fix: accept float, coerce to float, and validate range — never assume int from rating APIs.
pythonnormalizer/event_model.py
"""
Canonical FeedbackEvent — the unified schema all connectors produce.
Every field has a documented reason for its type choice.
"""
from __future__ import annotations
import uuid
from datetime import datetime, timezone
from typing import Literal, Optional
from pydantic import BaseModel, Field, field_validator, model_validator

Platform = Literal["google", "yelp", "doordash", "ubereats", "tripadvisor"]

VALID_LOCATION_IDS = {
    "cb-mission", "cb-castro", "cb-soma", "cb-tenderloin",
    "cb-richmond", "cb-sunset", "cb-oakland", "cb-berkeley",
    "cb-sj-downtown", "cb-sj-willow", "cb-sunnyvale", "cb-fremont",
}

class FeedbackEvent(BaseModel):
    # ── Identity ────────────────────────────────────────────────
    event_id:        str = Field(default_factory=lambda: str(uuid.uuid4()))
    source_event_id: str                  # platform's own review ID
    source_platform: Platform

    # ── Location ─────────────────────────────────────────────────
    location_id:     str
    location_name:   Optional[str] = None

    # ── Review content ───────────────────────────────────────────
    review_text:     str
    # float not int — Google Maps API v3 returns floats like 4.6
    star_rating:     float
    review_date:     datetime
    reviewer_name:   Optional[str] = None
    reviewer_id:     Optional[str] = None
    language:        str = "en"

    # ── Delivery metadata ────────────────────────────────────────
    ingested_at:     datetime = Field(default_factory=lambda: datetime.now(timezone.utc))
    raw_payload:     Optional[str] = None   # original JSON, for audit/reprocessing
    schema_version:  str = "1.0"

    @field_validator("star_rating")
    @classmethod
    def validate_rating(cls, v: float) -> float:
        if not 1.0 <= v <= 5.0:
            raise ValueError(f"star_rating must be 1.0–5.0, got {v}")
        # Normalize to 1 decimal place for consistency
        return round(v, 1)

    @field_validator("review_text")
    @classmethod
    def validate_text(cls, v: str) -> str:
        v = v.strip()
        if len(v) < 3:
            raise ValueError("review_text too short (min 3 chars)")
        if len(v) > 10_000:
            # Truncate, don't reject — some platforms allow very long reviews
            return v[:10_000]
        return v

    @field_validator("location_id")
    @classmethod
    def validate_location(cls, v: str) -> str:
        if v not in VALID_LOCATION_IDS:
            raise ValueError(f"Unknown location_id: {v}. Check VALID_LOCATION_IDS.")
        return v

    @field_validator("review_date")
    @classmethod
    def validate_date(cls, v: datetime) -> datetime:
        # Reject future dates — data pipeline bug indicator
        now = datetime.now(timezone.utc)
        if v.tzinfo is None:
            v = v.replace(tzinfo=timezone.utc)
        if v > now:
            raise ValueError(f"review_date {v} is in the future — likely a parsing bug")
        return v

    def to_avro_dict(self) -> dict:
        """Serialize to Avro-compatible dict (datetimes as ISO strings)."""
        d = self.model_dump()
        d["review_date"]  = self.review_date.isoformat()
        d["ingested_at"]  = self.ingested_at.isoformat()
        return d

03
Polling Connectors

Google Maps & Yelp: Rate-Limited Polling with Exponential Backoff

Polling connectors must survive API rate limits, network timeouts, partial failures, and daily quota exhaustion — all without losing a single review or duplicating it on restart.

🔥
Production Issue #2: API Rate Limit Cascade Without a token bucket or leaky bucket, a burst of reviews at midnight (restaurant closes → customers review) triggers simultaneous requests across all 12 locations → 429 Too Many Requests from Google → connector crashes → entire batch lost. Fix: tenacity with exponential backoff + jitter, per-connector rate state persisted across restarts.
⚠️
Production Issue #3: Cursor State Lost on Restart Every polling connector needs to remember where it stopped — the last page_token or since_timestamp. If this state lives only in memory, a crash means re-fetching from the beginning → duplicates in Kafka. Fix: persist cursor state to Redis or a simple JSON file on disk, read it on startup.
pythonconnectors/base_connector.py
"""
BaseConnector: shared logic for all polling connectors.
Handles rate limiting, retry with backoff+jitter, cursor persistence,
and DLQ routing. Every platform connector extends this.
"""
import json, time, hashlib
from abc import ABC, abstractmethod
from pathlib import Path
from typing import Iterator, Optional
import structlog
from tenacity import (
    retry, stop_after_attempt, wait_exponential_jitter,
    retry_if_exception_type, before_sleep_log
)
import requests
from normalizer.event_model import FeedbackEvent

log = structlog.get_logger(__name__)

class BaseConnector(ABC):
    PLATFORM: str          # subclass must declare e.g. "google"
    REQUESTS_PER_MINUTE: int = 30

    def __init__(self, config: dict, cursor_dir: str = ".cursors"):
        self.config     = config
        self.cursor_dir = Path(cursor_dir)
        self.cursor_dir.mkdir(exist_ok=True)
        self.session    = requests.Session()
        self._min_interval = 60.0 / self.REQUESTS_PER_MINUTE
        self._last_request_time = 0.0

    # ── Cursor persistence ──────────────────────────────────────
    def _cursor_path(self, location_id: str) -> Path:
        return self.cursor_dir / f"{self.PLATFORM}_{location_id}.json"

    def load_cursor(self, location_id: str) -> dict:
        path = self._cursor_path(location_id)
        if path.exists():
            return json.loads(path.read_text())
        return {"since_timestamp": "2024-01-01T00:00:00Z", "page_token": None}

    def save_cursor(self, location_id: str, cursor: dict) -> None:
        path = self._cursor_path(location_id)
        # Atomic write: write to temp file, then rename (prevents corrupt cursor)
        tmp = path.with_suffix(".tmp")
        tmp.write_text(json.dumps(cursor, indent=2))
        tmp.rename(path)

    # ── Rate limiting ───────────────────────────────────────────
    def _rate_limit_wait(self) -> None:
        elapsed = time.monotonic() - self._last_request_time
        gap     = self._min_interval - elapsed
        if gap > 0:
            time.sleep(gap)
        self._last_request_time = time.monotonic()

    # ── HTTP with retry ─────────────────────────────────────────
    @retry(
        retry=retry_if_exception_type((requests.HTTPError, requests.Timeout)),
        stop=stop_after_attempt(5),
        wait=wait_exponential_jitter(initial=2, max=120, jitter=10),
        before_sleep=before_sleep_log(log, log.warning),
    )
    def _get(self, url: str, **kwargs) -> requests.Response:
        self._rate_limit_wait()
        resp = self.session.get(url, timeout=30, **kwargs)
        if resp.status_code == 429:
            # Respect Retry-After header if provided
            retry_after = int(resp.headers.get("Retry-After", 60))
            log.warning("rate_limited", platform=self.PLATFORM, retry_after=retry_after)
            time.sleep(retry_after)
            resp.raise_for_status()   # triggers tenacity retry
        resp.raise_for_status()
        return resp

    # ── Dedup fingerprint ───────────────────────────────────────
    def _fingerprint(self, source_event_id: str, location_id: str) -> str:
        raw = f"{self.PLATFORM}|{source_event_id}|{location_id}"
        return hashlib.sha256(raw.encode()).hexdigest()

"""
Google Maps Places API v3 Connector.
Endpoint: places/{place_id}/reviews
Quota: 2,500 requests/day on free tier; 25,000 with billing enabled.
Pagination via pageToken; cursor persisted to disk between runs.
"""
import json
from typing import Iterator
from datetime import datetime, timezone
from pydantic import ValidationError
import structlog
from connectors.base_connector import BaseConnector
from normalizer.event_model import FeedbackEvent

log = structlog.get_logger(__name__)

# location_id → Google place_id mapping
LOCATION_PLACE_IDS = {
    "cb-mission":    "ChIJ7cv00DwsDogRAMDACa2m4K8",
    "cb-castro":     "ChIJIQBpAG2ahYAR_6128GcTUEo",
    "cb-soma":       "ChIJYXlyIr-Aj4ARpkXvtnbCreo",
    "cb-tenderloin": "ChIJ0RhLU9OAhYARNGWBkiM9jLw",
    # ... all 12 locations
}

class GoogleMapsConnector(BaseConnector):
    PLATFORM              = "google"
    REQUESTS_PER_MINUTE   = 20    # conservative: 2500/day ÷ 12 locations
    BASE_URL              = "https://places.googleapis.com/v1"

    def __init__(self, config: dict):
        super().__init__(config)
        self.api_key = config["google_api_key"]
        self.session.headers.update({
            "X-Goog-Api-Key":    self.api_key,
            "X-Goog-FieldMask":  "reviews,nextPageToken",
            "Content-Type":      "application/json",
        })

    def _parse_review(self, raw: dict, location_id: str) -> FeedbackEvent | None:
        try:
            return FeedbackEvent(
                source_event_id = raw["name"],   # "places/{id}/reviews/{review_id}"
                source_platform = "google",
                location_id     = location_id,
                location_name   = raw.get("authorAttribution", {}).get("displayName"),
                review_text     = raw.get("text", {}).get("text", ""),
                # Google returns float (e.g. 4.0) — BaseModel handles coercion
                star_rating     = float(raw["rating"]),
                review_date     = datetime.fromisoformat(
                    raw["publishTime"].replace("Z", "+00:00")
                ),
                reviewer_name   = raw.get("authorAttribution", {}).get("displayName"),
                reviewer_id     = raw.get("authorAttribution", {}).get("uri"),
                raw_payload     = json.dumps(raw),
            )
        except (ValidationError, KeyError) as e:
            log.error("google_parse_error", location=location_id,
                      error=str(e), raw_keys=list(raw.keys()))
            return None   # caller routes to DLQ

    def fetch_reviews(self, location_id: str) -> Iterator[FeedbackEvent]:
        place_id = LOCATION_PLACE_IDS.get(location_id)
        if not place_id:
            log.error("unknown_location", location_id=location_id); return

        cursor = self.load_cursor(location_id)
        page_token = cursor.get("page_token")
        fetched, skipped = 0, 0

        while True:
            params = {"pageSize": 50}
            if page_token:
                params["pageToken"] = page_token

            resp = self._get(
                f"{self.BASE_URL}/places/{place_id}/reviews", params=params
            )
            data       = resp.json()
            reviews    = data.get("reviews", [])
            page_token = data.get("nextPageToken")

            for raw in reviews:
                event = self._parse_review(raw, location_id)
                if event:
                    fetched += 1
                    yield event
                else:
                    skipped += 1

            # Persist cursor after every page — crash-safe incremental progress
            self.save_cursor(location_id, {
                "page_token": page_token,
                "last_fetched_at": datetime.now(timezone.utc).isoformat(),
                "fetched_total": fetched,
            })

            if not page_token:
                break

        log.info("google_fetch_complete", location=location_id,
                  fetched=fetched, skipped=skipped)

Webhook Connectors

DoorDash & Uber Eats: Webhook Receiver with HMAC Verification

Webhooks push data to you — which means attackers can too. Every incoming request must be HMAC-verified before processing. Replays, race conditions, and the idempotency key pattern are covered here.

🔥
Production Issue #4: Webhook Replay Attacks & Duplicates DoorDash retries webhook delivery if it doesn't receive a 2xx within 5 seconds. A slow Kafka producer causes a timeout → DoorDash retries → you receive the same review twice. Without idempotency keys checked against a seen-set (Redis), you get duplicate reviews in Kafka. Fix: HMAC verify → idempotency key check → then produce to Kafka → return 200 immediately.
pythonconnectors/webhook_receiver.py
"""
Webhook receiver for DoorDash and Uber Eats review events.
FastAPI endpoint: lightweight, async, responds 200 within milliseconds.
HMAC verification happens synchronously; Kafka produce is fire-and-forget.
Idempotency keys stored in Redis with 7-day TTL.
"""
import hashlib, hmac, json, time
from typing import Annotated
import redis
import structlog
from fastapi import FastAPI, Header, HTTPException, Request, BackgroundTasks
from pydantic import ValidationError
from confluent_kafka import Producer
from normalizer.event_model import FeedbackEvent

log = structlog.get_logger(__name__)
app = FastAPI(title="FIP Webhook Receiver")

class WebhookConfig:
    doordash_secret: str
    ubereats_secret: str
    kafka_brokers:   str
    redis_url:       str = "redis://localhost:6379"

cfg    = WebhookConfig()
rdb    = redis.from_url(cfg.redis_url, decode_responses=True)
producer = Producer({
    "bootstrap.servers": cfg.kafka_brokers,
    "enable.idempotence": True,   # producer-side exactly-once
    "acks": "all",                  # wait for all ISR replicas
    "retries": 5,
    "delivery.timeout.ms": 10_000,
})

def verify_hmac_sha256(body: bytes, signature: str, secret: str) -> bool:
    """
    Constant-time HMAC comparison. Never use == for signature comparison.
    Timing attacks can determine the correct signature byte-by-byte
    if you use regular string equality.
    """
    expected = hmac.new(
        secret.encode(), body, hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(expected, signature)

def is_duplicate(idempotency_key: str) -> bool:
    """Check + set atomically (SETNX). 7-day TTL matches DoorDash retry window."""
    redis_key = f"webhook:seen:{idempotency_key}"
    was_new   = rdb.set(redis_key, "1", nx=True, ex=604_800)  # 7 days
    return was_new is None   # None = key already existed = duplicate

def produce_to_kafka(event: FeedbackEvent) -> None:
    # Partition key = location_id → same location → same partition → ordered
    producer.produce(
        topic="feedback.raw.events",
        key=event.location_id,
        value=json.dumps(event.to_avro_dict()).encode(),
        on_delivery=lambda err, msg: log.error("kafka_delivery_failed", error=str(err)) if err else None
    )
    producer.poll(0)   # non-blocking flush of internal queue

@app.post("/webhooks/doordash")
async def doordash_webhook(
    request:        Request,
    background_tasks: BackgroundTasks,
    x_doordash_signature: Annotated[str | None, Header()] = None,
):
    body = await request.body()

    # ── Step 1: HMAC verification (synchronous — reject before any processing) ──
    if not x_doordash_signature:
        raise HTTPException(401, "Missing signature")
    if not verify_hmac_sha256(body, x_doordash_signature, cfg.doordash_secret):
        log.warning("hmac_verification_failed", source="doordash")
        raise HTTPException(401, "Invalid signature")

    payload          = json.loads(body)
    idempotency_key  = payload.get("event_id") or payload.get("review_id")

    # ── Step 2: Idempotency check (prevents duplicate on DoorDash retry) ──
    if not idempotency_key or is_duplicate(f"doordash:{idempotency_key}"):
        log.info("webhook_duplicate_skipped", key=idempotency_key)
        return {"status": "ok", "note": "duplicate"}   # still 200 — stops retry storm

    # ── Step 3: Parse & validate ──────────────────────────────────────────────
    try:
        event = FeedbackEvent(
            source_event_id = str(payload["review_id"]),
            source_platform = "doordash",
            location_id     = payload["store_id"],
            review_text     = payload["rating_details"].get("comment", ""),
            star_rating     = payload["rating_details"]["score"],
            review_date     = payload["created_at"],
            raw_payload     = body.decode(),
        )
    except (ValidationError, KeyError) as e:
        log.error("doordash_parse_failed", error=str(e))
        return {"status": "ok", "note": "routed_to_dlq"}   # still 200 — don't trigger retries

    # ── Step 4: Produce to Kafka in background (respond 200 FIRST) ───────────
    # Return 200 within milliseconds. DoorDash's 5s timeout is the enemy.
    background_tasks.add_task(produce_to_kafka, event)
    return {"status": "accepted"}

Kafka Producer

Idempotent Avro Producer with Dead Letter Queue Routing

The producer is the last gate before data enters the pipeline. Idempotent mode, delivery callbacks, and DLQ routing for messages that fail serialization or schema validation.

🔥
Production Issue #5: Silent Producer Failures Without a delivery callback, producer.produce() returns immediately and buffers the message internally. If Kafka is temporarily unavailable, the internal buffer fills up and messages are silently dropped with no error in your logs. Fix: always register a on_delivery callback, monitor the error metric, and alert on non-zero delivery failures.
pythonkafka/producer.py
"""
FIP Kafka Producer — wraps confluent-kafka with:
- Idempotent delivery (enable.idempotence=True)
- Avro serialization via Schema Registry
- Delivery callbacks with Prometheus metrics
- DLQ routing for unserializable messages
"""
import json
from typing import Optional, Callable
import structlog
from confluent_kafka import Producer, KafkaException
from confluent_kafka.schema_registry import SchemaRegistryClient
from confluent_kafka.schema_registry.avro import AvroSerializer
from confluent_kafka.serialization import SerializationContext, MessageField
from prometheus_client import Counter, Histogram
from normalizer.event_model import FeedbackEvent

log = structlog.get_logger(__name__)

# ── Prometheus metrics ──────────────────────────────────────────
EVENTS_PRODUCED = Counter(
    "fip_events_produced_total", "Events successfully delivered to Kafka",
    ["platform", "location_id"]
)
EVENTS_FAILED = Counter(
    "fip_events_failed_total", "Events that failed Kafka delivery",
    ["platform", "reason"]
)
PRODUCE_LATENCY = Histogram(
    "fip_produce_latency_seconds", "Time from produce() call to delivery callback"
)

class FIPProducer:
    def __init__(self, config: dict):
        self.topic    = config.get("topic", "feedback.raw.events")
        self.dlq      = config.get("dlq_topic", "feedback.dlq.events")

        sr_client = SchemaRegistryClient({"url": config["schema_registry_url"]})
        with open("kafka/schemas/feedback_event_v1.avsc") as f:
            schema_str = f.read()

        self.serializer = AvroSerializer(
            sr_client, schema_str,
            to_dict=lambda obj, ctx: obj.to_avro_dict()
        )

        self.producer = Producer({
            "bootstrap.servers":           config["kafka_brokers"],
            # ─── Idempotent mode: prevents duplicates from producer retries ───
            "enable.idempotence":           True,
            # ─────────────────────────────────────────────────────────────────
            "acks":                         "all",     # wait for all ISR replicas
            "retries":                       10,
            "max.in.flight.requests.per.connection": 5,
            "compression.type":             "snappy",
            "linger.ms":                    5,         # batch for 5ms for throughput
            "batch.size":                   65536,    # 64 KB batches
            "delivery.timeout.ms":          30_000,   # fail loudly after 30s
        })

    def _delivery_callback(self, err, msg, event: FeedbackEvent, start_time: float):
        latency = __import__('time').monotonic() - start_time
        if err:
            log.error("kafka_delivery_failed",
                      event_id=event.event_id, platform=event.source_platform,
                      error=str(err))
            EVENTS_FAILED.labels(platform=event.source_platform, reason=str(err)).inc()
        else:
            EVENTS_PRODUCED.labels(
                platform=event.source_platform,
                location_id=event.location_id
            ).inc()
            PRODUCE_LATENCY.observe(latency)

    def produce(self, event: FeedbackEvent) -> None:
        import time as _time
        start = _time.monotonic()
        try:
            serialized = self.serializer(
                event,
                SerializationContext(self.topic, MessageField.VALUE)
            )
            self.producer.produce(
                topic    = self.topic,
                key      = event.location_id,   # partition by location → ordered per location
                value    = serialized,
                on_delivery = lambda err, msg: self._delivery_callback(err, msg, event, start),
            )
            self.producer.poll(0)

        except Exception as e:
            log.error("produce_error", event_id=event.event_id, error=str(e))
            self._send_to_dlq(event, reason=str(e))

    def _send_to_dlq(self, event: FeedbackEvent, reason: str) -> None:
        """Poison messages go to DLQ with reason metadata. NEVER drop silently."""
        dlq_payload = {
            "original_event": event.to_avro_dict(),
            "dlq_reason":     reason,
            "dlq_timestamp":  __import__('datetime').datetime.utcnow().isoformat(),
        }
        self.producer.produce(
            topic = self.dlq,
            key   = event.location_id,
            value = json.dumps(dlq_payload).encode(),
        )
        EVENTS_FAILED.labels(platform=event.source_platform, reason="dlq").inc()

    def flush(self, timeout: float = 30.0) -> None:
        remaining = self.producer.flush(timeout)
        if remaining > 0:
            log.error("producer_flush_incomplete", remaining_messages=remaining)
            raise RuntimeError(f"{remaining} messages not delivered after {timeout}s flush")

Orchestration

Ingestion Runner: Wiring All Connectors with Graceful Shutdown

The main runner that schedules polling connectors, starts the webhook server, handles SIGTERM gracefully, and drains the Kafka producer buffer on shutdown.

⚠️
Production Issue #6: SIGTERM Without Graceful Drain Kubernetes sends SIGTERM before killing a pod. Without a signal handler, Python exits immediately, the Kafka producer buffer is discarded, and the last batch of reviews from that poll cycle is lost. Fix: register SIGTERM → set shutdown flag → wait for current poll to finish → call producer.flush(30) before exit.
pythonconnectors/ingestion_runner.py
"""
Main ingestion runner — orchestrates all polling connectors on a schedule.
Designed to run as a single long-lived process (Kubernetes Deployment).
Polling connectors run in a simple loop; webhook server runs in a thread.
"""
import signal, sys, time, threading
from datetime import datetime, timezone
import structlog
from confluent_kafka import Producer
from connectors.google_connector import GoogleMapsConnector
from connectors.yelp_connector import YelpConnector
from connectors.tripadvisor_connector import TripAdvisorConnector
from kafka.producer import FIPProducer
import uvicorn
from connectors.webhook_receiver import app as webhook_app

log = structlog.get_logger(__name__)

POLL_INTERVALS = {
    "google":      4 * 3600,    # 4 hours
    "yelp":        6 * 3600,    # 6 hours
    "tripadvisor": 12 * 3600,   # 12 hours
}

LOCATION_IDS = [
    "cb-mission", "cb-castro", "cb-soma", "cb-tenderloin",
    "cb-richmond", "cb-sunset", "cb-oakland", "cb-berkeley",
    "cb-sj-downtown", "cb-sj-willow", "cb-sunnyvale", "cb-fremont",
]

class IngestionRunner:
    def __init__(self, config: dict):
        self.config     = config
        self.producer   = FIPProducer(config)
        self._shutdown  = False
        self._last_poll: dict[str, float] = {}

        self.connectors = {
            "google":      GoogleMapsConnector(config),
            "yelp":        YelpConnector(config),
            "tripadvisor": TripAdvisorConnector(config),
        }

        # Graceful shutdown on SIGTERM (Kubernetes) or SIGINT (Ctrl+C)
        for sig in (signal.SIGTERM, signal.SIGINT):
            signal.signal(sig, self._handle_shutdown)

    def _handle_shutdown(self, signum, frame):
        log.info("shutdown_signal_received", signal=signum)
        self._shutdown = True

    def _should_poll(self, platform: str) -> bool:
        last     = self._last_poll.get(platform, 0)
        interval = POLL_INTERVALS[platform]
        return (time.monotonic() - last) >= interval

    def _poll_platform(self, platform: str, connector) -> None:
        log.info("poll_start", platform=platform)
        total = 0
        for location_id in LOCATION_IDS:
            if self._shutdown: break
            try:
                for event in connector.fetch_reviews(location_id):
                    self.producer.produce(event)
                    total += 1
            except Exception as e:
                # Log and continue — one bad location should not stop others
                log.error("poll_location_failed",
                           platform=platform, location=location_id, error=str(e))

        self._last_poll[platform] = time.monotonic()
        log.info("poll_complete", platform=platform, events_produced=total)

    def _start_webhook_server(self) -> threading.Thread:
        t = threading.Thread(
            target=uvicorn.run,
            kwargs={"app": webhook_app, "host": "0.0.0.0", "port": 8080, "log_level": "warning"},
            daemon=True,
        )
        t.start()
        log.info("webhook_server_started", port=8080)
        return t

    def run(self) -> None:
        self._start_webhook_server()
        log.info("ingestion_runner_started")

        while not self._shutdown:
            for platform, connector in self.connectors.items():
                if self._shutdown: break
                if self._should_poll(platform):
                    self._poll_platform(platform, connector)
            time.sleep(30)   # check every 30s whether any connector is due

        # ── Graceful drain before exit ──────────────────────────────
        # CRITICAL: without this, last batch is lost on SIGTERM
        log.info("draining_kafka_producer")
        self.producer.flush(timeout=30.0)
        log.info("shutdown_complete")

if __name__ == "__main__":
    from dotenv import load_dotenv
    import os
    load_dotenv()
    config = {
        "kafka_brokers":        os.getenv("KAFKA_BROKERS", "localhost:9092"),
        "schema_registry_url":  os.getenv("SCHEMA_REGISTRY_URL", "http://localhost:8081"),
        "google_api_key":       os.getenv("GOOGLE_API_KEY"),
        "yelp_api_key":         os.getenv("YELP_API_KEY"),
        "doordash_secret":      os.getenv("DOORDASH_WEBHOOK_SECRET"),
        "ubereats_secret":      os.getenv("UBEREATS_WEBHOOK_SECRET"),
    }
    IngestionRunner(config).run()

🎯 Interview Q&A — Senior Data / AI Engineer
Q: How do you design a multi-source ingestion pipeline that handles different API contracts?
Define a canonical event schema first — in our case a Pydantic model called FeedbackEvent — and make every connector responsible for mapping its platform's schema to that canonical form. This isolates platform-specific parsing behind a connector boundary: if Google changes their API, only GoogleMapsConnector changes, not the Kafka producer or the downstream lakehouse. The Pydantic model enforces invariants (rating range, minimum text length, no future dates) at the boundary so corrupt data is caught at ingestion, not discovered weeks later in an analytics dashboard.
Q: How do you prevent duplicates from webhook platforms that retry on timeout?
Two layers: first, always respond 200 within the platform's timeout window (DoorDash is 5 seconds). We do HMAC verification synchronously, then return 200, then produce to Kafka in a background task. Second, we maintain an idempotency key store in Redis using SETNX with a TTL that exceeds the platform's retry window (7 days for DoorDash). If the same event_id arrives twice, the second SETNX returns nil, we return 200 without producing, and the duplicate is dead. This pattern handles both timeout-driven retries and any manual re-sends from the platform.
Q: How do you handle rate limits across multiple API sources without a central coordinator?
Each polling connector maintains its own rate state — a monotonic timestamp of the last request and a minimum inter-request interval derived from the platform's quota. For Google at 2,500 requests/day across 12 locations, that's ~208 per location per day, so we set 20 requests/minute as a conservative cap. The BaseConnector's _rate_limit_wait() method enforces this locally. We pair it with tenacity retry logic that respects the platform's Retry-After header when a 429 is returned. For bursty conditions, we add jitter to the retry wait so all 12 location pollers don't wake up and hammer the API in lockstep after a rate limit event.
Q: What's your strategy for crash recovery in a polling connector?
Every page of results gets its cursor (page_token or since_timestamp) persisted to disk via an atomic write — write to a .tmp file, then rename — before we process the next page. On restart, we load the cursor and resume from the last known good position. This gives us at-most-once data loss on crash (we might re-fetch the current page) which we handle with dedup in the Silver layer. We intentionally avoid storing cursor in memory only. Rename is the key: it's atomic on every major filesystem, so there's no window where the file is partially written or missing.
