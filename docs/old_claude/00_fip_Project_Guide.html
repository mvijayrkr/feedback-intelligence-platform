Key architecture decisions
Why LangGraph over bare LangChain?
The Business Analyst Agent needs to call multiple sub-agents and combine results with conditional routing. LangGraph models this as an explicit stateful graph with edges and retry logic — LangChain alone doesn't handle multi-agent state cleanly. Stronger production story, stronger interview story.
Why Qdrant over pgvector?
pgvector is simpler but Qdrant gives hybrid search (dense + BM25) natively, per-collection tenant isolation, payload filtering, and horizontal scale. At 100K+ vectors per tenant, pgvector query performance degrades. Qdrant scales independently of RDS.
Why spaCy + OpenAI for NLP, not just OpenAI?
spaCy handles entity extraction and PII detection locally — no API cost, no latency, 10K docs/sec. OpenAI handles nuanced topic classification where reasoning matters. Right tool for each job = cost-efficient at pipeline scale = engineering judgment interviewers want to hear.
Why dbt instead of raw SQL for Gold layer?
dbt gives you data lineage, model testing, documentation, and incremental runs — the analytics engineering standard. Any EM role at a data-serious company expects dbt fluency. Raw SQL models don't scale with team size or model complexity.
Why hybrid search (dense + sparse)?
Pure vector search misses exact keyword matches (menu item names, location slugs, complaint codes). BM25 catches what embeddings miss. RRF re-ranking fuses both scores. This is how production RAG works at Elastic, Cohere, and Pinecone — not just demo RAG.
Why 7 LinkedIn days not 10?
Shorter series = higher completion rate = more algorithm amplification on LinkedIn. A 10-post series loses momentum. Each of these 7 posts is also denser — "I built a full data platform in one post" reads more impressive than spreading it thin. Quality over quantity for positioning.
