🎉
The Film Set Analogy A film set uses fake storefronts, prop food, and painted backdrops — everything looks real on camera without the cost of building an actual city block. The UI mock generator is your film set: it produces realistic FastAPI response fixtures so React components can be built, styled, and tested against real-looking data without a running FLOCI backend. Swap the mock for the real API with one env variable change — VITE_USE_MOCK=false.

"""
FIP UI Mock Generator -- Phase 5
Generates realistic FastAPI response fixtures for React component development.
Reuses Phase 1 FIPDataGenerator for review text content.

Usage:
  python scripts/generate_ui_fixtures.py --endpoint dashboard
  python scripts/generate_ui_fixtures.py --endpoint ask --question "worst ratings?"
  python scripts/generate_ui_fixtures.py --endpoint alerts
  python scripts/generate_ui_fixtures.py --all   # write all fixtures to ui/src/mocks/
"""
import argparse, json, random
from datetime import datetime, timedelta, timezone

import sys; sys.path.insert(0, "..")
from scripts.data_generator import FIPDataGenerator   # Phase 1 generator

LOCATIONS = [
    ("cb-mission",    "Mission District"),
    ("cb-castro",     "Castro"),
    ("cb-soma",       "SoMa"),
    ("cb-tenderloin", "Tenderloin"),
    ("cb-richmond",   "Richmond"),
    ("cb-sunset",     "Sunset"),
    ("cb-oakland",    "Oakland"),
    ("cb-berkeley",   "Berkeley"),
    ("cb-sj-downtown","SJ Downtown"),
    ("cb-sj-willow",  "SJ Willow Glen"),
    ("cb-sunnyvale",  "Sunnyvale"),
    ("cb-fremont",    "Fremont"),
]

class FIPUIMockGenerator:
    def __init__(self, seed: int = 42):
        random.seed(seed)
        self.gen = FIPDataGenerator(seed=seed)

    def dashboard(self) -> dict:
        """GET /api/dashboard response fixture."""
        locations = []
        for loc_id, loc_name in LOCATIONS:
            avg  = round(random.uniform(2.8, 4.9), 2)
            prev = round(avg + random.uniform(-0.8, 0.6), 2)
            locations.append({
                "location_id":     loc_id,
                "location_name":   loc_name,
                "avg_rating_7d":   avg,
                "avg_rating_14d":  prev,
                "rating_delta":    round(avg - prev, 2),
                "review_count_7d": random.randint(8, 94),
                "alert":           (avg - prev) < -0.5,
            })
        return {
            "locations":          sorted(locations, key=lambda x: x["avg_rating_7d"]),
            "total_alerts":       sum(1 for l in locations if l["alert"]),
            "last_ingested_at":   datetime.now(timezone.utc).isoformat(),
            "kafka_lag_messages": random.randint(0, 12),
        }

    def ask_response(self, question: str = "Which location needs attention?") -> dict:
        """POST /api/ask response fixture -- mirrors Phase 4 lambda_handler output."""
        review = self.gen.generate_one()
        return {
            "answer": (
                f"Based on recent customer reviews, {review['location_name']} "
                f"shows the most concerning feedback pattern this week. "
                f"Recommended action: schedule a management visit and review "\span>
                f"staffing levels for peak hours."
            ),
            "intent":          "ALERT",
            "nodes_visited":   ["intent_classifier", "alert_detector", "synthesize"],
            "alert_triggered": True,
            "alert_details":   [{
                "location_id":  review["location_id"],
                "current_avg":  3.1,
                "prev_avg":     3.8,
                "drop_amount":  0.7,
            }],
            "sources": [
                {"location_id": review["location_id"], "review_date": review["review_date"], "score": 0.91},
            ],
            "latency_ms": random.randint(820, 3400),
        }

    def write_all(self, out_dir: str = "ui/src/mocks"):
        import os; os.makedirs(out_dir, exist_ok=True)
        fixtures = {
            "dashboard.json": self.dashboard(),
            "ask_response.json": self.ask_response(),
        }
        for name, data in fixtures.items():
            path = os.path.join(out_dir, name)
            with open(path, "w") as f: json.dump(data, f, indent=2)
            print(f"Written: {path}")

if __name__ == "__main__":

FastAPI Backend

FastAPI App: Health, Auth Middleware & Agent Proxy Endpoint

The FastAPI backend is a thin proxy between the React UI and the Phase 4 LangGraph agent. It handles JWT auth, rate limiting, request validation, and structured logging. The agent runs as a module-level singleton — same pattern as the Lambda handler in Phase 4.

🏠
The Hotel Concierge Analogy A hotel concierge never changes the bedsheets, cooks the food, or fixes the plumbing themselves. They receive guest requests, route them to the right department, and return the response. The FastAPI backend is the concierge: it receives manager questions from the React UI, routes them to the Phase 4 LangGraph agent (the kitchen, housekeeping, maintenance team), and returns the structured response. The concierge adds auth, rate limiting, and logging — responsibilities that don't belong inside the agent.
🔥
Production Issue #1: Agent Singleton Re-Init on Every Request Importing and calling build_graph() inside a FastAPI route handler initialises LangGraph, spaCy, and the Qdrant connection on every HTTP request — adding 5–10s of overhead. Fix: initialise the agent at FastAPI startup using @app.on_event("startup") and store it in app.state.agent. One init per Uvicorn worker process.
pythonapi/main.py
"""
FIP FastAPI backend -- Phase 5.
Thin proxy: Auth -> Validate -> Agent invoke -> Return structured response.
Agent initialised at startup (one init per uvicorn worker process).
"""
import os, time
from contextlib import asynccontextmanager
from fastapi import FastAPI, Depends, HTTPException, Request
from fastapi.middleware.cors import CORSMiddleware
from prometheus_client import Counter, Histogram, make_asgi_app
import structlog

log = structlog.get_logger(__name__)

REQUEST_COUNT = Counter("fip_http_requests_total", "HTTP requests", ["method", "endpoint", "status"])
REQUEST_LATENCY = Histogram("fip_http_latency_seconds", "HTTP latency", ["endpoint"])

@asynccontextmanager
async def lifespan(app: FastAPI):
    # ── Startup: init agent ONCE per uvicorn worker ──────────────
    from agent.handler import lambda_handler   # Phase 4 handler
    app.state.agent_handler = lambda_handler
    log.info("agent_handler_ready", env=os.getenv("ENV", "local"))
    yield
    # ── Shutdown (graceful) ───────────────────────────────────────
    log.info("shutdown")

app = FastAPI(title="FIP API", version="1.0.0", lifespan=lifespan)

app.add_middleware(CORSMiddleware,
    allow_origins=["http://localhost:3000", os.getenv("UI_ORIGIN", "*")],
    allow_methods=["*"], allow_headers=["*"], allow_credentials=True)

# Prometheus metrics endpoint
app.mount("/metrics", make_asgi_app())

@app.middleware("http")
async def log_requests(request: Request, call_next):
    start = time.monotonic()
    response = await call_next(request)
    elapsed = time.monotonic() - start
    REQUEST_COUNT.labels(request.method, request.url.path, response.status_code).inc()
    REQUEST_LATENCY.labels(request.url.path).observe(elapsed)
    log.info("http_request", method=request.method,
             path=request.url.path, status=response.status_code,
             latency_ms=round(elapsed*1000))
    return response

@app.get("/health")
async def health():
    return {"status": "ok", "env": os.getenv("ENV","local"), "phase": "5"}

pythonapi/routers/ask.py
from fastapi import APIRouter, HTTPException, Request
from pydantic import BaseModel, Field

router = APIRouter()

class AskRequest(BaseModel):
    question:        str = Field(..., min_length=3, max_length=500)
    session_id:      str = Field(default="web-session")
    location_filter: str | None = None

class AskResponse(BaseModel):
    answer:          str
    intent:          str | None
    nodes_visited:   list[str]
    alert_triggered: bool
    alert_details:   list[dict]
    sources:         list[dict]
    latency_ms:      int | None = None

@router.post("/ask", response_model=AskResponse)
async def ask_agent(body: AskRequest, request: Request):
    handler = request.app.state.agent_handler
    import time; t0 = time.monotonic()
    result = handler({
        "session_id":     body.session_id,
        "question":       body.question,
        "location_filter": body.location_filter,
    }, context=None)
    if result.get("error"):
        raise HTTPException(500, detail=result["error"])
    return AskResponse(
        answer=result["answer"], intent=result.get("intent"),
        nodes_visited=result.get("nodes_visited",[]),
        alert_triggered=result.get("alert_triggered",False),
        alert_details=result.get("alert_details",[]),
        sources=result.get("sources",[]),
        latency_ms=int((time.monotonic()-t0)*1000),
    )

FastAPI Analytics & Dashboard Routes

Dashboard Endpoint (DuckDB Gold) + Analytics Trend Endpoint

The dashboard and analytics endpoints query Phase 2 dbt Gold Parquet via DuckDB read-only — same pattern as Phase 4 analytics_query_node. These endpoints bypass the agent for pure data retrieval: no LLM call, just fast SQL on Parquet. Under 200ms on FLOCI.

🍶
The Supermarket Self-Checkout Analogy A supermarket has cashiers (the agent) for complex situations and self-checkout lanes (direct DuckDB reads) for simple, fast operations. Asking "show me today's ratings for all locations" doesn't need an LLM — it's a simple SQL aggregation. Self-checkout is faster, cheaper, and never hallucinated. Only route to the full agent (cashier) when you need natural-language understanding.
🔥
Production Issue #2: No Caching on Dashboard Endpoint = N × DuckDB Scans 12 browser tabs open on the dashboard → 12 simultaneous DuckDB scans of Gold S3 Parquet → S3 cost spikes and p99 latency hits 8 seconds. Fix: Redis cache on GET /api/dashboard with a 60-second TTL. Dashboard data is never more fresh than the last dbt run (typically hourly), so 60s cache is free money.
pythonapi/routers/dashboard.py
import os, json, duckdb, hashlib
from fastapi import APIRouter
import redis, structlog

router = APIRouter()
log    = structlog.get_logger(__name__)
_cache = redis.from_url(os.getenv("REDIS_URL", "redis://localhost:6379"))
GOLD   = os.getenv("GOLD_PARQUET_PATH", "s3a://fip-curry-bowl-local-gold/fct_location_ratings/")

def gold_query(sql: str) -> list[dict]:
    # Exact same DuckDB in-memory pattern as Phase 4 analytics_query_node
    con = duckdb.connect(":memory:")
    con.execute("INSTALL httpfs; LOAD httpfs;")
    con.execute(f"""SET s3_endpoint='{os.getenv("DUCKDB_S3_ENDPOINT","s3.amazonaws.com")}';
                    SET s3_access_key_id='{os.getenv("DUCKDB_S3_ACCESS_KEY","")}';
                    SET s3_secret_access_key='{os.getenv("DUCKDB_S3_SECRET_KEY","")}';""")
    rel  = con.execute(sql)
    cols = [d[0] for d in rel.description]
    return [dict(zip(cols, r)) for r in rel.fetchall()]

@router.get("/dashboard")
async def dashboard():
    # 60-second Redis cache -- dashboard data is dbt-run fresh, not realtime
    cached = _cache.get("fip:dashboard")
    if cached: return json.loads(cached)

    rows = gold_query(f"""
        WITH cur AS (SELECT location_id, ROUND(AVG(avg_rating),2) avg7,
                     SUM(review_count) cnt
                     FROM read_parquet('{GOLD}**/*.parquet')
                     WHERE review_date >= CURRENT_DATE - INTERVAL '7 days'
                     GROUP BY location_id),
             prv AS (SELECT location_id, ROUND(AVG(avg_rating),2) avg14
                     FROM read_parquet('{GOLD}**/*.parquet')
                     WHERE review_date BETWEEN CURRENT_DATE-INTERVAL '14 days'
                                           AND CURRENT_DATE-INTERVAL '7 days'
                     GROUP BY location_id)
        SELECT c.location_id, c.avg7, p.avg14,
               ROUND(c.avg7 - p.avg14, 2) delta, c.cnt
        FROM cur c LEFT JOIN prv p USING(location_id)
        ORDER BY c.avg7 ASC""")

    result = {
        "locations":    rows,
        "total_alerts": sum(1 for r in rows if (r.get("delta") or 0) < -0.5),
    }
    _cache.setex("fip:dashboard", 60, json.dumps(result))
    log.info("dashboard_queried", locations=len(rows))
    return result

@router.get("/analytics/{location_id}")
async def location_trend(location_id: str):
    rows = gold_query(f"""
        SELECT review_date, avg_rating, review_count
        FROM read_parquet('{GOLD}**/*.parquet')
        WHERE location_id = '{location_id}'
        ORDER BY review_date DESC LIMIT 30""")
    return {"location_id": location_id, "trend": rows}

React Dashboard Home

Dashboard Component: Live Ratings Grid + Alert Banner + Zustand State

The React entry screen. Zustand manages global state (no Redux boilerplate). React Query handles API fetching with auto-refresh every 60 seconds. Alert banner fires when any location has a negative rating delta from the dashboard endpoint.

📷
The MRI Control Room Analogy The MRI machine (backend, agents, data pipeline) does complex invisible work. The radiologist's control room (React Dashboard) shows the results on clean displays. Zustand is the control room's state board: one source of truth visible to every screen. React Query is the automatic refresh cycle: the control room updates the display every 60 seconds without the radiologist pressing a button, the same way the dashboard re-fetches from the FastAPI backend on a fixed interval.
⚠
Production Issue #3: useEffect Fetch on Every Render = API Storm A common React mistake: fetching data inside useEffect([]) in every component that needs the dashboard data. 6 components → 6 simultaneous GET /api/dashboard calls on page load. Fix: React Query with a shared query key — all components subscribe to the same cached response. Only one network call per 60-second window regardless of how many components consume the data.

jsxui/src/pages/Dashboard.jsx
import { useQuery } from '@tanstack/react-query'
import { useDashboardStore } from '../stores/dashboardStore'
import axios from 'axios'

const API = import.meta.env.VITE_API_URL || 'http://localhost:8000'

function RatingBadge({ delta }) {
  const colour = delta < -0.5 ? 'text-red-400 bg-red-900/30'
               : delta <  0   ? 'text-yellow-400 bg-yellow-900/30'
               :                  'text-emerald-400 bg-emerald-900/30'
  return (
    <span className={`text-xs font-bold px-2 py-0.5 rounded-full ${colour}`}>
      {delta >= 0 ? '+' : ''}{delta?.toFixed(2)} △
    </span>
  )
}

export default function Dashboard() {
  const { setLocations } = useDashboardStore()

  // React Query: shared cache key, auto-refetch every 60s
  const { data, isLoading } = useQuery({
    queryKey: ['dashboard'],
    queryFn:  () => axios.get(`${API}/api/dashboard`).then(r => r.data),
    refetchInterval: 60_000,   // poll every 60s (dbt run cadence)
    staleTime:       55_000,   // don't refetch if data is <55s old
    onSuccess: (d) => setLocations(d.locations, d.total_alerts),
  })

  if (isLoading) return <div className="p-8 text-slate-400">Loading...</div>

  return (
    <div className="p-6 space-y-6">
      // Alert banner -- shows when any location has dropped >0.5 stars
      {data?.total_alerts > 0 && (
        <div className="bg-red-900/30 border border-red-500/40 rounded-lg p-4 flex gap-3">
          <span className="text-2xl">⚠</span>
          <div>
            <p className="font-bold text-red-300">
              {data.total_alerts} location{data.total_alerts > 1 ? 's' : ''} need attention
            </p>
            <p className="text-sm text-red-400">
              Rating dropped >0.5 stars vs last week. Go to Alerts tab for details.
            </p>
          </div>
        </div>
      )}

      <div className="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4">
        {data?.locations.map((loc) => (
          <div key={loc.location_id}
               className="bg-slate-800 border border-slate-700 rounded-xl p-4 hover:border-teal-500/40 transition cursor-pointer">
            <div className="flex justify-between items-start mb-2">
              <h3 className="font-semibold text-white">{loc.location_id.replace('cb-','')}</h3>
              <RatingBadge delta={loc.delta} />
            </div>
            <p className="text-3xl font-black text-teal-400 font-mono">{loc.avg7}</p>
            <p className="text-xs text-slate-500 mt-1">{loc.cnt} reviews this week</p>
          </div>
        ))}
      </div>
    </div>
  )
}

Ask Manager Screen

Natural-Language Question Box: Stream Agent Response to UI

The centrepiece of the UI. Managers type a question; the React component POST /api/ask to the FastAPI backend, which invokes the Phase 4 LangGraph agent and streams the response back. Shows intent, nodes visited, alert status, and source citations — all from the agent state.

🌀
The Mission Control Analogy NASA Mission Control asks "what is the spacecraft's trajectory?" and gets a structured answer: current speed, heading, fuel level, and recommended action — not raw telemetry numbers. The Ask Manager screen is Mission Control: the manager types a natural-language question and gets a structured answer with intent label, nodes visited (the diagnostic trail), alert status, and cited review sources — not raw Qdrant vectors or SQL results.
🔥
Production Issue #4: No Loading State = Manager Thinks It Crashed The Phase 4 agent takes 2–8 seconds to respond. A bare fetch() with no loading indicator leaves the manager staring at a frozen input box, assuming the app crashed. Fix: set isAsking: true in Zustand immediately on submit, show a pulsing "Thinking..." badge with the nodes visited updating in real-time (server-sent events or polling the agent status).
jsxui/src/pages/AskManager.jsx
import { useState } from 'react'
import { useMutation } from '@tanstack/react-query'
import { useDashboardStore } from '../stores/dashboardStore'
import axios from 'axios'

const API = import.meta.env.VITE_API_URL || 'http://localhost:8000'

const INTENT_COLOURS = {
  SUMMARY: 'bg-blue-900/40 text-blue-300',
  TREND:   'bg-purple-900/40 text-purple-300',
  COMPARE: 'bg-amber-900/40 text-amber-300',
  ALERT:   'bg-red-900/40 text-red-300',
}

export default function AskManager() {
  const [question, setQuestion] = useState('')
  const { askResult, isAsking, setAskResult, setAsking } = useDashboardStore()

  const mutation = useMutation({
    mutationFn: (q) =>
      axios.post(`${API}/api/ask`, { question: q, session_id: 'web-ui' })
           .then(r => r.data),
    onMutate:   () => setAsking(true),
    onSuccess:  (data) => { setAskResult(data); setAsking(false) },
    onError:    () => setAsking(false),
  })

  const handleSubmit = (e) => {
    e.preventDefault()
    if (question.trim()) mutation.mutate(question)
  }

  return (
    <div className="p-6 space-y-6 max-w-3xl">
      <form onSubmit={handleSubmit} className="space-y-3">
        <textarea
          value={question} onChange={e => setQuestion(e.target.value)}
          placeholder="Ask anything: 'Which location needs attention this week?'"
          className="w-full bg-slate-800 border border-slate-700 rounded-xl p-4 text-white resize-none h-28 focus:border-teal-500 focus:outline-none"
        />
        <button type="submit" disabled={isAsking || !question.trim()}
          className="bg-teal-500 text-black font-bold px-6 py-2.5 rounded-lg hover:bg-teal-400 disabled:opacity-40 disabled:cursor-not-allowed">
          {isAsking ? '🧠 Thinking...' : 'Ask Agent'}
        </button>
      </form>

      {isAsking && (
        <div className="bg-slate-800 border border-slate-700 rounded-xl p-5 animate-pulse">
          <div className="h-4 bg-slate-700 rounded w-3/4 mb-2"></div>
          <div className="h-4 bg-slate-700 rounded w-1/2"></div>
        </div>
      )}

      {askResult && !isAsking && (
        <div className="bg-slate-800 border border-slate-700 rounded-xl p-5 space-y-4">
          // Intent + nodes trail
          <div className="flex flex-wrap gap-2 items-center">
            <span className={`text-xs font-bold px-2.5 py-1 rounded-full ${INTENT_COLOURS[askResult.intent]}`}>
              {askResult.intent}
            </span>
            {askResult.nodes_visited.map(n => (
              <span key={n} className="text-xs bg-slate-700 text-slate-300 px-2 py-0.5 rounded font-mono">
                {n}
              </span>
            ))}
          </div>

          {askResult.alert_triggered && (
            <div className="bg-red-900/30 border border-red-500/40 rounded-lg p-3">
              <p className="text-red-300 font-bold text-sm">⚠ Alert Detected</p>
              {askResult.alert_details.map(a => (
                <p key={a.location_id} className="text-xs text-red-400 mt-1">
                  {a.location_id}: dropped {a.drop_amount} stars (was {a.prev_avg}, now {a.current_avg})
                </p>
              ))}
            </div>
          )}

          <p className="text-slate-200 leading-relaxed">{askResult.answer}</p>

          {askResult.sources?.length > 0 && (
            <div className="text-xs text-slate-500">
              Sources: {askResult.sources.map(s => s.location_id).join(', ')}
            </div>
          )}

          <p className="text-xs text-slate-600">
            Latency: {askResult.latency_ms}ms
          </p>
        </div>
      )}
    </div>
  )
}

Location Analytics Chart

Recharts 30-Day Rating Trend + React Router Drill-Down

Clicking a location card on Dashboard navigates to /location/:id. This page fetches the 30-day rating trend from GET /api/analytics/{location_id} and renders a Recharts LineChart. The chart data comes from Phase 2 dbt Gold — exact same Parquet files Phase 4 agents query for analytics.

📈
The Stock Ticker Analogy A stock price chart shows the same underlying data as the raw trade log — but visualised so a human can instantly spot trends, drops, and recoveries without reading thousands of numbers. The Recharts LineChart does the same for rating data: the same avg_rating values that live in the dbt Gold Parquet become a visual trend line that a manager can interpret in 2 seconds. The chart doesn't add new data — it adds comprehension.
⚠
Production Issue #5: Recharts Renders Empty on Missing Data Points If a location has zero reviews on a specific day (e.g. the store was closed), that date is missing from the Parquet partition. Recharts renders a gap in the line — a broken chart that looks like a bug. Fix: fill missing dates with the previous day's value in the FastAPI route before returning to the frontend. Use DuckDB's GENERATE_SERIES to produce a full date spine and LEFT JOIN the review data against it.
jsxui/src/pages/LocationTrend.jsx
import { useParams } from 'react-router-dom'
import { useQuery } from '@tanstack/react-query'
import { LineChart, Line, XAxis, YAxis, CartesianGrid,
         Tooltip, ReferenceLine, ResponsiveContainer } from 'recharts'
import axios from 'axios'

const API = import.meta.env.VITE_API_URL || 'http://localhost:8000'

export default function LocationTrend() {
  const { id } = useParams()

  const { data, isLoading } = useQuery({
    queryKey: ['trend', id],
    queryFn:  () => axios.get(`${API}/api/analytics/${id}`).then(r => r.data),
    staleTime: 300_000,  // 5 min cache -- trend doesn't change fast
  })

  const chartData = (data?.trend || [])
    .slice().reverse()  // chronological order for chart
    .map(r => ({
      date:   r.review_date?.slice(5),  // "MM-DD" format
      rating: parseFloat(r.avg_rating),
      count:  r.review_count,
    }))

  if (isLoading) return <div className="p-8 text-slate-400">Loading trend...</div>

  return (
    <div className="p-6 space-y-4">
      <h2 className="text-xl font-bold text-white">
        {id} — 30-Day Rating Trend
      </h2>
      <div className="bg-slate-800 border border-slate-700 rounded-xl p-6">
        <ResponsiveContainer width="100%" height={280}>
          <LineChart data={chartData}>
            <CartesianGrid strokeDasharray="3 3" stroke="#334155" />
            <XAxis dataKey="date" tick={{fill:'#64748b', fontSize:11}} />
            <YAxis domain={[1, 5]} tick={{fill:'#64748b', fontSize:11}} />
            <Tooltip
              contentStyle={{background:'#1e293b', border:'1px solid #334155'}}
              labelStyle={{color:'#94a3b8'}}
            />
            // Alert threshold line at 3.5
            <ReferenceLine y={3.5} stroke="#ef4444" strokeDasharray="4 4"
              label={{value:'Alert threshold', fill:'#ef4444', fontSize:10}} />
            <Line type="monotone" dataKey="rating"
                  stroke="#14b8a6" strokeWidth={2} dot={false}
                  activeDot={{r:5, fill:'#14b8a6'}} />
          </LineChart>
        </ResponsiveContainer>
      </div>
    </div>
  )
}

App Shell + Routing + Auth Guard

React Router Layout Shell, JWT Auth Middleware, Vite Config

Wire all screens together in a React Router shell with a persistent sidebar nav. JWT auth guard protects all routes — managers log in with their store credentials. Vite proxy config routes /api/* to FastAPI in dev, eliminating CORS issues locally.

🏠
The Shopping Mall Analogy A shopping mall has a main entrance (auth guard), a directory board at the front (sidebar nav), and individual stores inside (screen components). You can't enter a store without passing through the entrance. React Router is the mall's internal corridor: it routes shoppers (users) to the right store (screen) based on the URL. The sidebar is the directory board — always visible, always current.
jsxui/src/App.jsx
import { BrowserRouter, Routes, Route, Navigate } from 'react-router-dom'
import { QueryClient, QueryClientProvider } from '@tanstack/react-query'
import Dashboard   from './pages/Dashboard'
import AskManager  from './pages/AskManager'
import LocationTrend from './pages/LocationTrend'
import Alerts      from './pages/Alerts'
import Sidebar     from './components/Sidebar'
import Login       from './pages/Login'
import { useAuthStore } from './stores/authStore'

const qc = new QueryClient({ defaultOptions: { queries: { retry: 1 } } })

function AuthGuard({ children }) {
  const token = useAuthStore(s => s.token)
  return token ? children : <Navigate to="/login" replace />
}

export default function App() {
  return (
    <QueryClientProvider client={qc}>
      <BrowserRouter>
        <Routes>
          <Route path="/login" element={<Login />} />
          <Route path="/*" element={
            <AuthGuard>
              <div className="flex min-h-screen bg-slate-900">
                <Sidebar />
                <main className="flex-1 overflow-auto">
                  <Routes>
                    <Route path="/"              element={<Navigate to="/dashboard" replace />} />
                    <Route path="/dashboard"    element={<Dashboard />} />
                    <Route path="/ask"          element={<AskManager />} />
                    <Route path="/location/:id" element={<LocationTrend />} />
                    <Route path="/alerts"       element={<Alerts />} />
                  </Routes>
                </main>
              </div>
            </AuthGuard>
          } />
        </Routes>
      </BrowserRouter>
    </QueryClientProvider>
  )
}

I built a full-stack dashboard for a restaurant AI platform. React + FastAPI + LangGraph agents on a shared Docker stack. Three things that surprised me and what I learned from each: 🔥 The agent re-init that killed performance My first FastAPI route handler called build_graph() (LangGraph initialisation) on every HTTP request. That's 5-10 seconds of startup every time a manager clicked anything. FastAPI's lifespan() context manager runs once at startup and once at shutdown. Initialise the agent there, store it in app.state. Every subsequent request reuses the same object. The first manager question takes 2.8 seconds. Every subsequent one in the same Uvicorn process takes 400ms. Module-level singletons are the right pattern for expensive objects. This applies whether you're writing FastAPI, Lambda handlers, or Flask apps. 🔥 The useEffect fetch storm Three components all needed dashboard data. Each did its own useEffect([], []) fetch. Three components = three simultaneous GET /api/dashboard calls on every page load. Multiply by 12 concurrent users = 36 DuckDB S3 scans per page refresh. React Query with a shared query key solves this completely: all three components subscribe to the same cached response. One network call. Zero code duplication. Automatic staleness-based refetch. This is the reason to use React Query instead of bare fetch() calls. 🔥 The chart gap that looked like a bug A location with zero reviews on a Wednesday had no row in the Parquet partition for that date. Recharts rendered a gap in the rating line. Every time I showed a manager, they pointed at the gap and asked "is the data broken?" 🎯 Interview Q&A — Engineering Manager / Senior Full-Stack / Senior AI Engineer
Q: How does the Phase 5 UI enforce the Phase 0 hard safety boundary without duplicating it?
It doesn't need to — that's the point of the layered architecture. The React UI only calls FastAPI endpoints. FastAPI only calls the Phase 4 lambda_handler. The handler only invokes the LangGraph agent. The agent's analytics_query_node uses DuckDB in-memory (enforces read-only at driver level), and the synthesize node's system prompt forbids SQL output (enforces at LLM level). The UI has no direct connection to Qdrant, Kafka, S3, or any database. If a manager somehow injected a malicious question, it would be classified by the intent_classifier, routed to a read-only DuckDB or RAG query, and the safety-bounded synthesize node would respond. The UI's only job is to display results and submit questions. Safety lives at Phase 4, not Phase 5.
Q: Why use React Query instead of just useEffect for API calls?
Three reasons. First, deduplication: multiple components that need the same data share one cached response via a shared query key. useEffect in each component creates N simultaneous network calls. Second, staleness management: React Query knows when data is stale and refetches automatically — you configure staleTime and refetchInterval declaratively, not imperatively with setInterval in useEffect. Third, loading and error states are standardised: every useQuery call returns the same {data, isLoading, error} shape, so components handle loading states consistently without ad-hoc boolean flags. The practical result: our dashboard refetches every 60 seconds, all three components that display location data share one cached response, and the loading skeleton is the same pattern everywhere. That would take 3x the code with useEffect.
Q: How do you ensure the FLOCI development environment produces the same behaviour as production?
The key is that we use the same Docker image in both environments — not just the same code. The Dockerfile is the build artifact, not the Python source code. The docker-compose.phase5.yml mounts source volumes for hot-reload in dev, but the final Docker image baked for ECS Fargate contains the same pinned requirements.txt, same Python version, and same OS base layer. The only differences between local and prod are environment variables: in FLOCI, QDRANT_HOST=qdrant (Docker network name), DUCKDB_S3_ENDPOINT=http://localstack:4566, and LAMBDA_INVOKE_MODE=local. In prod, these point to the real services. The Terraform ECS task definition uses the same variable pattern: var.qdrant_host and var.duckdb_s3_endpoint. The engineer changes one .tfvars file, not the application code.
Q: How would you handle real-time rating alerts — not polling every 60 seconds?
For true real-time, we'd add a Server-Sent Events (SSE) endpoint to FastAPI: GET /api/alerts/stream. The FastAPI endpoint keeps an open connection and pushes events whenever the alert_detector identifies a new drop. On the React side, useEffect opens an EventSource connection to this endpoint and updates the Zustand alerts store when a message arrives — no polling, no React Query interval. The alert_detector would run on a Kafka consumer that subscribes to a new Kafka topic (fip.alerts) published by a background worker that runs DuckDB checks every 5 minutes. This is intentionally Phase 6 infrastructure: in Phase 5 we prove the UI concept works with 60s polling; we only build real-time streaming when we know what the acceptable latency SLA actually is. Premature real-time infrastructure is expensive to operate and maintain.
