01

Business Requirement

Why this phase exists from the business perspective

Curry Bowl Express is a 12-location Indian restaurant chain in the Bay Area. The owner is losing customers — ratings are dropping, delivery complaints are rising, and there is no unified system to understand what is going wrong across locations. Before building any intelligent analysis, they need a production-grade technical foundation that can ingest, store, and process feedback at scale.

🎯 Business Problem
  • 12 locations with no unified feedback view
  • Ratings declining but no root cause analysis
  • Feedback scattered across Google, Reddit, delivery apps, surveys
  • Managers making decisions without data
  • No way to identify which location needs urgent attention
  • No mechanism to turn feedback into operational actions
✅ What Phase 0 Delivers
  • Foundation that all later phases build upon
  • AWS-parity local environment via FLOCI
  • Infrastructure-as-code with Terraform
  • Standard event contract for all feedback sources
  • Dummy data strategy before live integrations
  • Security, observability, and tenant isolation from day one

Business Questions This Platform Will Answer

📍 Location

Which location is failing? Which needs urgent attention today?

📉 Root Cause

Why is rating dropping? Which issue is affecting revenue the most?

🔁 Pattern

Which feedback is repeated across sources? What themes recur weekly?

🍛 Menu

What are customers saying about biryani? Which item drives complaints?

📋 Action

What should the manager do this week? What next-best action should be created?

💰 Revenue

Which complaints are linked to repeat-customer churn and revenue drop?

makefile
# Makefile — one executable Phase 0 bootstrap flow
# Important: Terraform does not magically start FLOCI. The bootstrap target starts FLOCI,
# points AWS/Terraform to FLOCI, provisions AWS-shaped resources, creates Kafka topics,
# configures kubectl, then deploys Helm.

SHELL := /bin/bash
.ONESHELL:
.DEFAULT_GOAL := help

AWS_REGION ?= us-west-2
FLOCI_ENDPOINT ?= http://localhost:4566
TF_DIR := infra/terraform/envs/local-floci
HELM_CHART := infra/helm/feedback-platform
NAMESPACE := feedback
CLUSTER_NAME := feedback-eks-local

.PHONY: help bootstrap floci-start tf-init tf-plan tf-apply secrets-inject k8s-config topics helm-install verify tf-destroy clean

help:
	@echo "Phase 0 commands:"
	@echo "  make bootstrap     # start FLOCI → terraform apply → topics → kubectl → helm → verify"
	@echo "  make tf-destroy    # destroy local Terraform resources"
	@echo "  make clean         # stop FLOCI"

bootstrap: floci-start tf-init tf-apply secrets-inject k8s-config topics helm-install verify
	@echo "✅ Phase 0 local AWS-parity foundation is running"

floci-start:
	floci start
	eval $$(floci env)
	aws --endpoint-url $(FLOCI_ENDPOINT) sts get-caller-identity
	aws --endpoint-url $(FLOCI_ENDPOINT) s3 ls || true

tf-init:
	cd $(TF_DIR)
	terraform init -upgrade

tf-plan:
	cd $(TF_DIR)
	terraform plan

tf-apply:
	cd $(TF_DIR)
	terraform apply -auto-approve
	terraform output

secrets-inject:
	aws --endpoint-url $(FLOCI_ENDPOINT) secretsmanager put-secret-value --secret-id /feedback/local/openai_api_key --secret-string "sk-dummy-local-key-replace-in-prod" || true
	aws --endpoint-url $(FLOCI_ENDPOINT) secretsmanager put-secret-value --secret-id /feedback/local/jwt_secret --secret-string "local-jwt-secret-$$(openssl rand -hex 16)" || true
	aws --endpoint-url $(FLOCI_ENDPOINT) secretsmanager put-secret-value --secret-id /feedback/local/db_password --secret-string "local-db-password-$$(openssl rand -hex 8)" || true

k8s-config:
	aws --endpoint-url $(FLOCI_ENDPOINT) eks update-kubeconfig --region $(AWS_REGION) --name $(CLUSTER_NAME)
	kubectl get nodes

topics:
	chmod +x scripts/create_topics.sh
	KAFKA_BOOTSTRAP_SERVERS=$${KAFKA_BOOTSTRAP_SERVERS:-localhost:9092} ./scripts/create_topics.sh

helm-install:
	kubectl create namespace $(NAMESPACE) --dry-run=client -o yaml | kubectl apply -f -
	helm upgrade --install feedback-platform $(HELM_CHART) 	  --namespace $(NAMESPACE) --create-namespace 	  -f $(HELM_CHART)/values-local.yaml 	  --wait --timeout 120s

verify:
	aws --endpoint-url $(FLOCI_ENDPOINT) s3 ls
	aws --endpoint-url $(FLOCI_ENDPOINT) secretsmanager list-secrets --query 'SecretList[].Name'
	kubectl get pods -n $(NAMESPACE)
	kubectl get svc -n $(NAMESPACE)

tf-destroy:
	cd $(TF_DIR)
	terraform destroy -auto-approve

clean:
	floci stop
05

FLOCI Setup

Install and configure FLOCI as your local AWS emulator

ℹ️
What is FLOCI?

FLOCI is an open-source local AWS stack that emulates AWS APIs locally. It lets you run the same Terraform AWS provider and AWS CLI commands locally that you'll use in production — the only change is the endpoint URL.

Step-by-Step FLOCI Installation

  1. 1

    Install Prerequisites

    Docker Desktop, AWS CLI v2, Terraform v1.6+, Helm v3, kubectl

  2. 2

    Install FLOCI

    Install via the FLOCI CLI installer or Homebrew

  3. 3

    Start FLOCI

    Run floci start — this starts the local AWS-emulation container stack

  4. 4

    Set Environment Variables

    Run eval $(floci env) to point AWS CLI to FLOCI endpoints

  5. 5

    Verify AWS CLI Works Against FLOCI

    Run aws sts get-caller-identity and aws s3 ls

  6. 6

    Point Terraform Provider to FLOCI

    Set floci_endpoint = "http://localhost:4566" in terraform.tfvars

  7. 7

    Initialize Terraform

    Run terraform init in infra/terraform/envs/local-floci/

  8. 8

    Apply Infrastructure

    Run terraform apply to provision S3, MSK, RDS, Secrets, EKS locally

  9. 9

    Configure kubectl for FLOCI EKS

    Run aws eks update-kubeconfig --region us-west-2 --name feedback-eks-local

  10. 10

    Deploy Helm Charts

    Run helm upgrade --install feedback-platform ... with local values file

🔧
Execution Flow

The execution flow is: install tools → start FLOCI → export FLOCI AWS endpoint variables → run Terraform against FLOCI → inject local secret values → configure kubectl from FLOCI EKS → create Kafka topics → deploy Helm → verify pods/services.

Schema-First Design

Every source — Google, Reddit, DoorDash, surveys, QR, manager notes, voice — must produce a valid FeedbackEvent before Kafka publish. Reject at the source. Never let malformed data into the pipeline.

from __future__ import annotations
import uuid
from datetime import datetime
from enum import Enum
from typing import Any
from pydantic import BaseModel, Field, field_validator, model_validator


class FeedbackSource(str, Enum):
    GOOGLE_REVIEWS   = "google_reviews"
    REDDIT_MENTIONS  = "reddit_mentions"
    DOORDASH         = "doordash_reviews"
    UBEREATS         = "ubereats_reviews"
    SURVEY           = "survey_responses"
    QR_FEEDBACK      = "qr_feedback"
    MANAGER_NOTES    = "manager_notes"
    VOICE_TRANSCRIPT = "voice_transcripts"


class CustomerSegment(str, Enum):
    NEW     = "new"
    REPEAT  = "repeat_customer"
    LOYALTY = "loyalty_member"
    UNKNOWN = "unknown"


class OrderType(str, Enum):
    DELIVERY = "delivery"
    PICKUP   = "pickup"
    DINE_IN  = "dine_in"
    UNKNOWN  = "unknown"


class ProcessingStage(str, Enum):
    RAW      = "raw"
    CLEANED  = "cleaned"
    NLP      = "nlp_enriched"
    EMBEDDED = "embedded"


class FeedbackEvent(BaseModel):
    """
    Universal feedback event.
    All 7 sources normalise into this before Kafka publish.
    Invalid events → DLQ immediately. Never corrupt the pipeline.
    """
    # Identity
    event_id:         str            = Field(default_factory=lambda: str(uuid.uuid4()))
    tenant_id:        str            = Field(..., description="Multi-tenant isolation key")
    source:           FeedbackSource
    source_review_id: str
    location_id:      str            = Field(..., description="Normalised slug: san_jose")

    # Content
    rating:      float | None = Field(None, ge=0.0, le=5.0)
    review_text: str
    review_ts:   datetime
    ingested_at: datetime     = Field(default_factory=datetime.utcnow)

    # Segmentation
    customer_segment: CustomerSegment = CustomerSegment.UNKNOWN
    order_type:       OrderType       = OrderType.UNKNOWN
    language:         str             = "en"

    # Business context
    business_impact_hint: str | None = None
    upvotes:              int | None = None   # Reddit
    platform:             str | None = None   # doordash/ubereats

    # Pipeline metadata
    stage:          ProcessingStage = ProcessingStage.RAW
    schema_version: str             = "1.0.0"

    # Raw passthrough for Bronze audit layer
    raw_payload: dict[str, Any] = Field(default_factory=dict)

    @field_validator("tenant_id")
    @classmethod
    def tenant_not_empty(cls, v: str) -> str:
        v = v.strip().lower()
        if not v:
            raise ValueError("tenant_id must not be empty")
        return v

    @field_validator("review_text")
    @classmethod
    def text_min_length(cls, v: str) -> str:
        v = v.strip()
        if len(v) < 3:
            raise ValueError("review_text too short")
        return v[:10_000]

    @field_validator("location_id")
    @classmethod
    def location_normalise(cls, v: str) -> str:
        return v.strip().lower().replace(" ", "_")

    @model_validator(mode="after")
    def cross_field_checks(self) -> "FeedbackEvent":
        if self.source in (FeedbackSource.DOORDASH, FeedbackSource.UBEREATS):
            if self.order_type == OrderType.UNKNOWN:
                self.order_type = OrderType.DELIVERY
        return self

    class Config:
        use_enum_values = True


class EnrichedFeedbackEvent(FeedbackEvent):
    """Phase 2+ NLP fields added after nlp-worker processes the event."""
    sentiment:       str   | None = None   # positive / neutral / negative
    sentiment_score: float | None = None   # -1.0 to +1.0
    topics:          list[str]    = Field(default_factory=list)
    severity:        int   | None = None   # 1 (low) to 5 (critical)
    pii_masked:      bool         = False
    masked_text:     str   | None = None   # PII-stripped for vector indexing
    embedding_id:    str   | None = None   # Qdrant point ID (Phase 3)

async def require_tenant(
    x_tenant_id: Annotated[str | None, Header()] = None
) -> str:
    """
    FastAPI dependency injected on every protected route.
    Usage: @app.get("/data", dependencies=[Depends(require_tenant)])

    if not x_tenant_id:

    if tenant not in _ALLOWED_TENANTS:

    return tenant

import json, logging, os, time
from datetime import datetime, timezone
from typing import Any


class FIPLogger:
    """
    Structured JSON logger — all services must use this.
    All log lines machine-parseable by CloudWatch Logs Insights.
    """
    def __init__(self, service: str):
        self.service = service
        self._logger = logging.getLogger(service)
        if not self._logger.handlers:
            h = logging.StreamHandler()
            h.setFormatter(logging.Formatter("%(message)s"))
            self._logger.addHandler(h)
        self._logger.setLevel(os.getenv("LOG_LEVEL", "INFO").upper())

    def _emit(self, level: str, event: str, **fields: Any):
        entry = {
            "ts":      datetime.now(timezone.utc).isoformat(),
            "level":   level,
            "service": self.service,
            "event":   event,
            **fields,
        }
        getattr(self._logger, level.lower())(json.dumps(entry))

    def info(self,  e: str, **kw): self._emit("INFO",    e, **kw)
    def warn(self,  e: str, **kw): self._emit("WARNING",  e, **kw)
    def error(self, e: str, **kw): self._emit("ERROR",   e, **kw)

    def timed(self, event: str, **kw):
        """Context manager that auto-logs event with latency_ms."""
        return _TimedCtx(self, event, kw)


class _TimedCtx:
    def __init__(self, log, event, fields):
        self.log, self.event, self.fields = log, event, fields
    def __enter__(self):
        self._t = time.monotonic(); return self
    def __exit__(self, exc_type, *_):
        ms = round((time.monotonic() - self._t) * 1000, 2)
        self.log.info(self.event, latency_ms=ms,
                      status="error" if exc_type else "success",
                      **self.fields)

Interview Q&A

Phase 0 talking points for Engineering Manager and Senior AI/Data interviews. Edit in Edit Mode.

💬 "How do you approach local development for cloud-native AI systems?"
I insist on AWS-parity in local development using FLOCI — not just "something that runs locally." The same Terraform AWS provider modules that create S3 buckets, MSK Kafka clusters, RDS instances, and Secrets Manager entries locally are the exact modules that run in production. The only change is a provider endpoint config. This means developers run real terraform apply commands from day one against local infrastructure. By Phase 6 when we deploy to real AWS, the team has executed the same workflow dozens of times. No cloud shock, no drift, no "it works on my machine."
💬 "How do you enforce multi-tenancy and security from day one, not as an afterthought?"
Multi-tenancy is enforced at every layer as a schema constraint, not just an API check. Every Kafka event carries a tenant_id field — validated by Pydantic before publish. Every FastAPI route uses a tenant dependency that validates the X-Tenant-Id header. PostgreSQL uses row-level security by tenant_id. Qdrant uses per-tenant collection namespaces. For LLM agents specifically, no agent generates raw SQL that touches the database — all DB access goes through typed, read-only tool functions I define and control. This prevents prompt injection from a malicious review text turning into a DROP TABLE. Defense in depth across every layer.
💬 "Why did you choose LangGraph over just LangChain for the agent framework?"
The Business Analyst Agent needs to call both the Data Analyst Agent and the RAG Agent in parallel, wait for both results, and then reason over the combined output to generate root cause and recommendations. LangGraph models this as an explicit stateful graph with nodes, edges, and conditional routing — LangChain alone doesn't give you this structure cleanly. LangGraph also handles retries and partial failures at the graph level, not just the individual call level. For production multi-agent systems where the orchestration logic is as important as the individual agent logic, LangGraph is the right choice.
💬 "What makes this production-grade versus a proof of concept?"
Five specific decisions from Phase 0. First: no hardcoded secrets anywhere — everything through Secrets Manager from the first line. Second: manual Kafka offset commits — we control exactly when a message is acknowledged, enabling proper retry-then-DLQ patterns. Third: structured JSON logging from every service — machine-parseable by CloudWatch Logs Insights in production. Fourth: readiness and liveness probes on every pod, non-root security context, read-only filesystem. Fifth: DLQ from day one — not added later when events start failing in production. Systems that lack any of these spend months retrofitting them while debugging incidents. I'd rather spend the time now.
💬 "As an Engineering Manager, how does this project demonstrate your technical depth?"
The foundation phase is where I invest technical depth upfront so the team moves fast in every phase after. The Makefile with a single make bootstrap command, the shared logging and auth libraries, the standard Helm values template, the FeedbackEvent schema as a typed contract — these are all force multipliers. A new engineer can run make bootstrap on day one and have a running local stack. Infrastructure decisions made here — FLOCI, Terraform module structure, Kafka topic naming, S3 bucket layout — persist across all 7 phases without revisiting them. That's how you build a team that ships features fast without accumulating technical debt on the foundation.
