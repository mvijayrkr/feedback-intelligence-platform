03

Install Developer Toolchain

Install only laptop-side CLI tools. Kafka brokers run inside FLOCI MSK, not locally.

Important separation: FLOCI provisions the runtime services such as EKS and MSK. Your Mac only needs CLI tools like Terraform, AWS CLI, kubectl, Helm, and optionally Kafka command-line tools to connect to FLOCI MSK.
bash — developer tools only
# macOS developer prerequisites
# These are tools installed on your laptop.
# They do NOT start Kubernetes or Kafka locally.
brew install   awscli   terraform   helm   kubectl   jq   yq   make   uv

# Optional: Kafka CLI tools only.
# This is NOT the Kafka runtime for the project.
# Bootstrap works without it — make topics skips topic creation with a warning.
# Install only when you want kafka-topics.sh for local MSK admin commands.
brew install kafka

# FLOCI install
brew tap floci/tap
brew install floci

# Alternative FLOCI install
# curl -fsSL https://install.floci.io | bash

# Confirm developer tool versions
floci --version
aws --version
terraform version
helm version
kubectl version --client
jq --version
yq --version
uv --version

# Optional Kafka CLI verification.
# If this works, the CLI is installed.
# The Kafka brokers are still provisioned later as FLOCI MSK.
kafka-topics.sh --version || true
Do not start local Kafka: Do not run brew services start kafka, Docker Compose Kafka, ZooKeeper, or Redpanda for this project. Producers and consumers must connect to the FLOCI MSK bootstrap endpoint created by Terraform.

Makefile

This is the final Phase 0 execution controller.

makefile — Makefile
# Makefile — Phase 0 FLOCI Foundation with EKS + MSK
# This is a tutorial/course Makefile. It assumes you already created every file shown in the guide.

SHELL := /bin/bash
.DEFAULT_GOAL := help

AWS_REGION ?= us-west-2
FLOCI_ENDPOINT ?= http://localhost:4566
TF_DIR := infra/terraform/envs/local-floci
HELM_CHART := infra/helm/feedback-platform
NAMESPACE := feedback
CLUSTER_NAME := fip-local-eks
MSK_CLUSTER_NAME := fip-local-msk

.PHONY: help doctor bootstrap floci-start tf-check tf-init tf-plan tf-apply k8s-config topics helm-install verify tf-destroy clean

help:
	@echo "Phase 0 commands:"
	@echo "  make doctor      # verify required local tools"
	@echo "  make bootstrap   # FLOCI -> Terraform(EKS/MSK/S3/Secrets/IAM/etc) -> kubectl -> topics -> Helm -> verify"
	@echo "  make tf-plan     # preview Terraform resources"
	@echo "  make tf-destroy  # destroy local Terraform resources"
	@echo "  make clean       # stop FLOCI"

doctor:
	@command -v floci >/dev/null || (echo "❌ floci not installed"; exit 1)
	@command -v aws >/dev/null || (echo "❌ aws cli not installed"; exit 1)
	@command -v terraform >/dev/null || (echo "❌ terraform not installed"; exit 1)
	@command -v kubectl >/dev/null || (echo "❌ kubectl not installed"; exit 1)
	@command -v helm >/dev/null || (echo "❌ helm not installed"; exit 1)
	@command -v kafka-topics.sh >/dev/null || echo "⚠️ kafka-topics.sh not installed. Kafka runs in FLOCI MSK; install Kafka CLI only if you want topic/admin commands locally: brew install kafka"
	@echo "✅ Required tools are installed"

bootstrap: doctor floci-start tf-check tf-init tf-apply k8s-config topics helm-install verify
	@echo "✅ Phase 0 FLOCI foundation completed: EKS + MSK + S3 + IAM + Secrets + Helm"

floci-start:
	floci start
	aws --endpoint-url $(FLOCI_ENDPOINT) sts get-caller-identity
	aws --endpoint-url $(FLOCI_ENDPOINT) s3 ls || true

tf-check:
	@test -f $(TF_DIR)/versions.tf || (echo "❌ Missing $(TF_DIR)/versions.tf"; exit 1)
	@test -f $(TF_DIR)/provider.tf || (echo "❌ Missing $(TF_DIR)/provider.tf"; exit 1)
	@test -f $(TF_DIR)/main.tf || (echo "❌ Missing $(TF_DIR)/main.tf"; exit 1)
	@test -f $(TF_DIR)/outputs.tf || (echo "❌ Missing $(TF_DIR)/outputs.tf"; exit 1)
	@echo "✅ Terraform files found"
	@ls -la $(TF_DIR)

tf-init:
	terraform -chdir=$(TF_DIR) init -upgrade

tf-plan:
	terraform -chdir=$(TF_DIR) plan

tf-apply:
	terraform -chdir=$(TF_DIR) apply -auto-approve
	terraform -chdir=$(TF_DIR) output || true

k8s-config:
	aws --endpoint-url $(FLOCI_ENDPOINT) eks update-kubeconfig --region $(AWS_REGION) --name $(CLUSTER_NAME)
	kubectl get nodes

topics:
	@if command -v kafka-topics.sh >/dev/null; then \
	  chmod +x scripts/create_topics.sh; \
	  ./scripts/create_topics.sh; \
	else \
	  echo "⚠️ Skipping topic creation because kafka-topics.sh is not installed."; \
	  echo "   Kafka runtime is FLOCI MSK. Install CLI only if needed: brew install kafka"; \
	fi

helm-install:
	kubectl create namespace $(NAMESPACE) --dry-run=client -o yaml | kubectl apply -f -
	helm lint ./$(HELM_CHART) -f $(HELM_CHART)/values-local.yaml
	helm upgrade --install feedback-platform ./$(HELM_CHART) \
	  --namespace $(NAMESPACE) --create-namespace \
	  -f $(HELM_CHART)/values-local.yaml \
	  --wait --timeout 180s

verify:
	@echo "🔎 S3 buckets"
	aws --endpoint-url $(FLOCI_ENDPOINT) s3 ls
	@echo "🔎 Secrets"
	aws --endpoint-url $(FLOCI_ENDPOINT) secretsmanager list-secrets --query 'SecretList[].Name'
	@echo "🔎 EKS cluster"
	aws --endpoint-url $(FLOCI_ENDPOINT) eks describe-cluster --region $(AWS_REGION) --name $(CLUSTER_NAME) --query 'cluster.name'
	@echo "🔎 MSK cluster"
	aws --endpoint-url $(FLOCI_ENDPOINT) kafka list-clusters --region $(AWS_REGION) --query 'ClusterInfoList[].ClusterName'
	@echo "🔎 Kubernetes pods"
	kubectl get pods -n $(NAMESPACE)
	@echo "🔎 Kafka topics"
	kafka-topics --list --bootstrap-server $$(terraform -chdir=$(TF_DIR) output -raw kafka_bootstrap_brokers 2>/dev/null || echo localhost:9092)

tf-destroy:
	terraform -chdir=$(TF_DIR) destroy -auto-approve

clean:
	floci stop

Kafka Topic Creation Script

This creates the reusable topics in FLOCI MSK for Phase 1 and beyond.

bash — scripts/create_topics.sh
#!/usr/bin/env bash
set -euo pipefail

TF_DIR="infra/terraform/envs/local-floci"
BOOTSTRAP_SERVERS="${KAFKA_BOOTSTRAP_SERVERS:-}"

if [[ -z "$BOOTSTRAP_SERVERS" ]]; then
  BOOTSTRAP_SERVERS="$(terraform -chdir="$TF_DIR" output -raw kafka_bootstrap_brokers 2>/dev/null || echo localhost:9092)"
fi

echo "Using Kafka bootstrap servers: $BOOTSTRAP_SERVERS"

TOPICS=(
  "feedback.raw.events"
  "feedback.validated.events"
  "feedback.bronze.events"
  "feedback.silver.events"
  "feedback.action.events"
  "feedback.dlq.events"
)

for topic in "${TOPICS[@]}"; do
  kafka-topics.sh \
    --bootstrap-server "$BOOTSTRAP_SERVERS" \
    --create \
    --if-not-exists \
    --topic "$topic" \
    --partitions 3 \
    --replication-factor 1
  echo "✅ topic ready: $topic"
done

kafka-topics.sh --bootstrap-server "$BOOTSTRAP_SERVERS" --list

Troubleshooting

Common problems and exact fixes.

Terraform says “No configuration files”

You ran Terraform before creating versions.tf, provider.tf, and main.tf, or your Makefile is old. Use terraform -chdir=$(TF_DIR) in every Terraform target.

Secrets list is empty

Use aws_secretsmanager_secret_version resources. Do not rely on CLI put-secret-value after Terraform.

kubectl localhost:8080 refused

The EKS cluster was not created or kubeconfig was not updated. Run make tf-apply, then make k8s-config.

kafka-topics.sh not found

Bootstrap still succeeds without it — make topics skips topic creation with a warning. Install Kafka CLI (brew install kafka) when you need to create or list FLOCI MSK topics locally.
