Course Rules

The corrected model for this documentation set

Rule 1: Every code file is created explicitly in the guide. No hidden generator script. No phase assumes files that were not already created.
Rule 2: Each phase extends the same repo. Nothing recreates infrastructure after Phase 0.
Rule 3: Run execution commands only after completing the “Files to Create” section in that phase.

05

Phase Index

Open each detailed build guide

Phase 0 — Foundation

Consumes: None. This is the first phase.

Adds: Creates the repo, Terraform, Helm, shared schema, logging, topics, secrets, and local AWS-parity foundation.

Produces: A running FLOCI-backed foundation that Phase 1 reuses.

Open Phase 0

Phase 1 — Data Ingestion

Consumes: Phase 0 foundation completed; FLOCI, S3, topics, env, shared schema exist.

Adds: Adds data generator, Kafka producer, ingestion worker, and DLQ pattern.

Produces: Raw feedback events available for Phase 2 cleaning.

Open Phase 1

Phase 2 — Data Platform

Consumes: Phase 1 has generated/published raw feedback events.

Adds: Adds Bronze cleaner, PII masker, NLP enrichment, Silver data, and dbt Gold model skeleton.

Produces: PII-safe Silver and Gold analytics models for Phase 3/4.

Open Phase 2

Phase 3 — RAG Layer

Consumes: Phase 2 has Silver feedback data.

Adds: Adds embedding worker, Qdrant Helm config, and RAG/hybrid search service.

Produces: Vector/RAG evidence layer for Phase 4 agents.

Open Phase 3

Phase 4 — Agents

Consumes: Phase 3 RAG service and Phase 2 Gold/Silver data exist.

Adds: Adds governed tools, RAG agent, business analyst agent, action/audit pattern.

Produces: Auditable agent backend for Phase 5 UI.

Open Phase 4

Phase 5 — Product UI

Consumes: Phase 4 agent backend exists.

Adds: Adds FastAPI APIs, React dashboard, chat/voice service placeholders, product surface.

Produces: A usable product front-end over governed agents and marts.

Open Phase 5

Phase 6 — Production Deploy

Consumes: Phases 0–5 work locally.

Adds: Adds GitHub Actions, prod Terraform provider, k6, monitoring/security deployment assets.

Produces: Production promotion path using the same repo/artifacts.

Open Phase 6
